API 同步到 Apifox 工具使用帮助

## 功能简介
与 Apifox 项目同步后端接口，支持自动解析接口增删改查和手动触发同步，字段说明使用中文展示。支持增量同步和全量更新。

## 命令说明

### 1. sync 同步命令
同步后端接口到 Apifox 项目

**使用方法：**
```
api-sync-to-apifox sync [参数]
```

**参数列表：**
- --apifox-project-id <ID> (必填): Apifox 项目 ID
- --apifox-api-key <API_KEY> (必填): Apifox API 密钥
- --source-type <swagger|code> (必填): 接口源类型
- --source-path <PATH> (必填): 源路径（Swagger URL 或代码目录）
- --framework <springboot|nodejs> (可选): 后端框架类型（当 source-type 为 code 时必填）
- --trigger-mode <auto|manual> (可选): 触发模式（默认: auto）
- --sync-mode <incremental|full> (可选): 同步模式（默认: incremental）
- --api-method <method> (可选): 单独接口同步的 HTTP 方法（GET/POST/PUT/DELETE）
- --api-path <path> (可选): 单独接口同步的路径

**示例：**
1. 从 Swagger 同步接口：
   ```
   api-sync-to-apifox sync --apifox-project-id 12345 --apifox-api-key abc123 --source-type swagger --source-path "https://api.example.com/v2/api-docs"
   ```

2. 从 Spring Boot 代码同步接口（增量同步）：
   ```
   api-sync-to-apifox sync --apifox-project-id 12345 --apifox-api-key abc123 --source-type code --source-path "./backend" --framework springboot --sync-mode incremental
   ```

3. 从 Node.js 代码全量同步接口：
   ```
   api-sync-to-apifox sync --apifox-project-id 12345 --apifox-api-key abc123 --source-type code --source-path "./api" --framework nodejs --sync-mode full
   ```

4. 单独同步 GET /api/users 接口：
   ```
   api-sync-to-apifox sync --apifox-project-id 12345 --apifox-api-key abc123 --source-type code --source-path "./backend" --framework springboot --api-method GET --api-path "/api/users"
   ```

5. 单独同步 POST /api/users 接口：
   ```
   api-sync-to-apifox sync --apifox-project-id 12345 --apifox-api-key abc123 --source-type code --source-path "./backend" --framework springboot --api-method POST --api-path "/api/users"
   ```

### 2. scan 扫描命令
扫描后端接口变更（不执行同步）

**使用方法：**
```
api-sync-to-apifox scan [参数]
```

**参数列表：**
- --source-type <swagger|code> (必填): 接口源类型
- --source-path <PATH> (必填): 源路径（Swagger URL 或代码目录）
- --framework <springboot|nodejs> (可选): 后端框架类型（当 source-type 为 code 时必填）
- --scan-type <all|changed> (可选): 扫描类型（默认: changed）

**示例：**
1. 扫描所有接口：
   ```
   api-sync-to-apifox scan --source-type code --source-path "./backend" --framework springboot --scan-type all
   ```

2. 只扫描变更的接口：
   ```
   api-sync-to-apifox scan --source-type code --source-path "./backend" --framework springboot --scan-type changed
   ```

3. 扫描 Swagger 文档接口：
   ```
   api-sync-to-apifox scan --source-type swagger --source-path "https://api.example.com/v2/api-docs"
   ```

## 同步模式说明

### 1. 增量同步（incremental）
- 默认模式
- 只同步代码变更过的接口
- 检测 git 中修改过的文件，只扫描和同步这些文件中的接口
- 输出接口变更统计（新增、更新、删除）
- 适用于开发过程中的日常同步

### 2. 全量更新（full）
- 同步所有接口，不检测代码变更
- 适用于项目初始化或重构后的完整同步
- 会同步所有接口，可能需要更长时间

## 源类型说明

### 1. Swagger 文档源（swagger）
- 从 Swagger/OpenAPI 文档同步接口
- 支持 JSON 和 YAML 格式
- 适用于已有完整文档的项目

### 2. 代码解析源（code）
- 直接解析源代码获取接口信息
- 支持 Spring Boot 和 Node.js 项目
- 自动格式化接口说明为中文
- 适用于文档不完整的项目

## 获取 Apifox 项目信息

### 1. 获取项目 ID
1. 登录 Apifox
2. 进入项目
3. 在项目设置中查看项目 ID

### 2. 获取 API 密钥
1. 登录 Apifox
2. 进入个人设置 → API 密钥
3. 创建或查看已有的 API 密钥

## 注意事项

1. 代码解析功能基于 git 变更检测，需要在 git 仓库中使用
2. Spring Boot 支持：
   - 识别 @RestController 注解的类
   - 识别 @GetMapping、@PostMapping、@PutMapping、@DeleteMapping 注解的方法
3. Node.js 支持：
   - 识别 app.get()、app.post()、app.put()、app.delete() 等路由方法
4. 接口字段说明会自动格式化为中文
5. 同步过程会显示详细的接口变更信息
