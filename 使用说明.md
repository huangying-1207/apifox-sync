# Apifox 同步工具使用说明（方法二）

## 快速开始

### 1. 安装工具
在您的项目根目录执行：

```bash
npm install file:path/to/api-sync-to-apifox-1.2.0.tgz --save-dev
```

### 2. 初始化配置
```bash
npx api-sync-to-apifox config init
```

### 3. 编辑配置文件
打开生成的 `.apifoxsync.json` 文件，填入您的 Apifox 项目信息：

```json
{
  "apifox-project-id": "您的项目ID",
  "apifox-api-key": "您的API密钥",
  "source-type": "code",
  "source-path": "./src",
  "framework": "springboot"
}
```

### 4. 扫描接口
```bash
npx api-sync-to-apifox scan
```

### 5. 同步接口
```bash
npx api-sync-to-apifox sync
```

## 添加到项目脚本

在您的 `package.json` 中添加以下脚本，方便使用：

```json
{
  "scripts": {
    "apifox:init": "api-sync-to-apifox config init",
    "apifox:scan": "api-sync-to-apifox scan",
    "apifox:sync": "api-sync-to-apifox sync"
  }
}
```

然后就可以这样使用：

```bash
npm run apifox:init
npm run apifox:scan
npm run apifox:sync
```

## 高级用法

### 使用命令行参数（覆盖配置文件）

```bash
# 从指定的 Spring Boot 代码同步
npx api-sync-to-apifox sync --apifox-project-id 12345 --apifox-api-key abc123 --source-type code --source-path "./src/main/java" --framework springboot

# 从 Swagger 文档同步
npx api-sync-to-apifox sync --apifox-project-id 12345 --apifox-api-key abc123 --source-type swagger --source-path "http://localhost:8080/v2/api-docs"

# 全量更新
npx api-sync-to-apifox sync --sync-mode full

# 手动触发模式
npx api-sync-to-apifox sync --trigger-mode manual
```

### CI/CD 集成

在 CI/CD 流程中使用（以 GitHub Actions 为例）：

```yaml
name: Apifox Sync

on:
  push:
    branches: [main]

jobs:
  sync:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Set up Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '16'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Sync to Apifox
        run: |
          cat > .apifoxsync.json << 'EOF'
          {
            "apifox-project-id": "${{ secrets.APIFOX_PROJECT_ID }}",
            "apifox-api-key": "${{ secrets.APIFOX_API_KEY }}",
            "source-type": "code",
            "source-path": "./src/main/java",
            "framework": "springboot"
          }
          EOF
          
          npx api-sync-to-apifox sync
```

## 获取 Apifox 项目信息

### 获取项目 ID

1. 登录 Apifox
2. 进入项目
3. 点击右上角的 "设置" 图标
4. 在项目设置中查看项目 ID

### 获取 API 密钥

1. 登录 Apifox
2. 点击右上角的头像，选择 "个人设置"
3. 在左侧菜单选择 "API 密钥"
4. 创建或查看已有的 API 密钥

## 故障排除

### 1. 权限问题

如果遇到权限问题，尝试以下方法：

```bash
# 修复权限
chmod +x node_modules/api-sync-to-apifox/src/index.js

# 重新安装
npm uninstall api-sync-to-apifox
npm install file:path/to/api-sync-to-apifox-1.2.0.tgz --save-dev
```

### 2. Node.js 版本兼容性

确保您使用的 Node.js 版本符合要求：

```bash
node --version
# 应该是 12.0 或更高版本
```

### 3. Git 环境问题

增量同步功能需要在 Git 仓库中使用：

```bash
git status
# 如果不是 git 仓库，需要初始化
git init && git add . && git commit -m "Initial commit"
```

## 支持的框架

### Spring Boot
- 识别 `@RestController` 注解的类
- 识别 `@GetMapping`、`@PostMapping`、`@PutMapping`、`@DeleteMapping` 注解的方法

### Node.js
- 识别 `app.get()`、`app.post()`、`app.put()`、`app.delete()` 等路由方法
- 支持 Express 框架

## 配置选项说明

| 配置项 | 必填 | 说明 | 可选值 | 默认值 |
|--------|------|------|--------|--------|
| apifox-project-id | ✔️ | Apifox 项目 ID | - | - |
| apifox-api-key | ✔️ | Apifox API 密钥 | - | - |
| source-type | ✔️ | 数据源类型 | swagger\|code | code |
| source-path | ✔️ | 源路径 | 文件/目录路径或 URL | ./src |
| framework | ✔️（当 source-type 为 code 时） | 后端框架类型 | springboot\|nodejs | springboot |
| sync-mode | ❌ | 同步模式 | incremental\|full | incremental |
| trigger-mode | ❌ | 触发模式 | auto\|manual | auto |
| scan-type | ❌ | 扫描类型 | all\|changed | changed |

## 常见问题

### 问：配置文件找不到怎么办？

答：工具会在以下位置查找配置文件：
1. 当前目录下的 `.apifoxsync.json`
2. `./.claude/apifoxsync.json`
3. `./config/apifoxsync.json`
4. 用户主目录下的 `.apifoxsync.json`

### 问：同步过程中出现网络错误怎么办？

答：确保您的网络可以正常访问 Apifox API。如果使用代理，请配置代理：

```bash
export https_proxy=http://proxy.example.com:8080
```

### 问：接口没有被正确识别怎么办？

答：
1. 确保代码中使用了标准的注解（Spring Boot）或路由方法（Node.js）
2. 检查 `source-path` 是否正确
3. 尝试使用 `--scan-type all` 参数扫描所有文件
